# 00_Входящие — обработка входящих файлов

Сюда кидаешь любой новый файл (PDF, MD, TXT, скрин, заметку), который нужно ввести в проект.

---

## Зачем этот подход

Файл, который просто лежит в папке — мёртвый груз. Файл, введённый в структуру — живой контекст, к которому агент обращается с минимумом токенов. Поэтому каждый новый файл сначала попадает сюда, а потом **осознанно раскладывается** по проекту.

---

## Протокол обработки (говоришь агенту)

```
Прочитай 00_СТАТУС.md.
В папке 00_Входящие/ лежит файл [имя].
Обработай его по протоколу.
```

**Агент выполняет 5 шагов:**

### Шаг 1 — Прочитать и классифицировать

Определить: что это за файл?

| Тип | Куда переедет |
|-----|---------------|
| Источник, статья, официальный отчёт, кейс | `01_источники/` |
| Гайд интервью, анкета, сырые ответы опроса | `02_эмпирика_сырьё/` |
| Диаграмма, BPMN, схема | `03_модели_экспорт/` |
| Черновик главы, заметки для написания | `04_текст_главы/` |
| Слайды, тезисы защиты | `05_презентация/` |
| Аналитика, deep research, выводы по теме | корень проекта (с номером) |

### Шаг 2 — Извлечь полезное

Ответить на вопросы:
- Что меняет в статусе проекта? → обновить `00_СТАТУС.md`
- Что добавляет к контексту проекта? → обновить `01_КОНТЕКСТ.md` (соответствующий раздел)
- Это новый источник для главы 1? → добавить в `01_источники/таблица_учёта_литературы.md`
- Это данные для эмпирики? → добавить описание в `02_эмпирика_сырьё/README.md`

### Шаг 3 — Переместить файл

Переместить из `00_Входящие/` в целевую папку с осмысленным именем:
- Формат: `тип_краткое_название_ГГГГ.расширение`
- Пример: `источник_dodo_eta_report_2024.md`, `интервью_клиент01_20260610.md`

### Шаг 4 — Обновить README.md проекта

Добавить строку в таблицу «Рабочие папки» с описанием нового файла.

### Шаг 5 — Написать 3 строки в этот файл (лог обработки)

```
| ГГГГ-ММ-ДД | имя_файла | куда переехал | что изменилось |
```
