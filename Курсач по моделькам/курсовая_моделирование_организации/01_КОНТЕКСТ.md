## 1. Суть проекта

**Дисциплина:** курсовой проект «Моделирование организации», ВШЭ, методичка 2-е изд. 2022.  
**Объект:** Додо Пицца / Dodo Brands — сеть пиццерий с доставкой, платформа Dodo IS.  
**Сфера:** доставка готовой еды в России.

### Структура отчёта

| Часть | Требования |
|-------|-----------|
| Введение | ≤ 3 стр.; актуальность, цель, задачи, структура, источники |
| **Глава 1** — организационные рамки | Только официальные факты. **Без анализа и выводов** |
| **Глава 2** — диагностика | Обоснование методов, эмпирика, дерево проблем, Ishikawa |
| **Глава 3** — модели | Landscape → BPMN AS IS/TO BE процесса #1 (доступность и ETA/SLA) + модель TO BE процесса #2 (рекламации), глоссарий, эффект |
| Список источников | ≥ 20, ГОСТ, часть на англ. |
| Оригинальность текста | ≥ 80% |
| Презентация | 12–15 слайдов, 10 мин; вся команда отвечает на вопросы |

## 2. Требования преподавателя

| Тезис | Следствие |
|-------|-----------|
| Методичка = минимум; семинар был под ДЗ | Курсач — **цельный продукт**, не цепочка домашек |
| **Ранжирование процессов обязательно** | Таблица кандидатов → `03_РАНЖИРОВАНИЕ.md` (сейчас 2 процесса) |
| TO BE выбираете сами, с аргументацией | TO BE = «**продукт, который купила бы компания**» |
| Не «7 ручных шагов → 5» | Нужна управленческая ценность, не косметика схемы |
| 2 консультации на **группу** | Нести **свою логику** (цель, границы, ранжирование) |
| Не «всё ли правильно в BPMN?» | Спрашивать: «Логичен ли выбор процесса при такой проблеме?» |
| Формальных checkpoint нет | Свой график; **вся команда** готовится к защите |

Полные ответы: `04_ПРЕПОД.md`. Две темы и план: `02_ВАРИАНТЫ_ТЕМ.md`, `07_ПЛАН_РАБОТЫ.md`.

## 3. Порядок работы

> Актуальный пошаговый план: **`07_ПЛАН_РАБОТЫ.md`**. Кратко:

1. Landscape + ранжирование **двух** процессов (`03_РАНЖИРОВАНИЕ.md`).
2. Глава 1 + источники.
3. Эмпирика → глава 2 (обе проблемы).
4. SIPOC → BPMN гл. 3 (#1 детальный, #2 упрощённый).
5. Сборка, презентация.

**Не делать:** BPMN до landscape; два несвязанных TO BE в **одном** BPMN.

---

## 4. Ограничения команды

- **Контакты:** только открытые источники + возможен опрос клиентов (нет инсайдеров в Додо).
- AS IS без инсайда → реконструкция + явные допущения + `[ТРЕБУЕТСЯ ИСТОЧНИК]`.
- TO BE = **предложение** улучшения, не утверждение «так уже работает у Додо».

## 6. Правила для ИИ-агента

1. **Не выдумывать** факты, цифры, регламенты Додо → `[ТРЕБУЕТСЯ ИСТОЧНИК]` или вопрос к пользователю.
2. **Глава 1** — только официальные факты, без анализа и выводов.
3. **Главы 2–3** — выводы опираются на материалы в `02_эмпирика_сырьё/` или явные данные пользователя.
4. **BPMN** — давать текстовый сценарий + элементы; финальная диаграмма в bpmn.io / ARIS / Bizagi.
5. **Не генерировать** главы «под копипаст» без тезисов автора.
6. **Библиография** — ГОСТ, без вымышленных изданий и DOI.
