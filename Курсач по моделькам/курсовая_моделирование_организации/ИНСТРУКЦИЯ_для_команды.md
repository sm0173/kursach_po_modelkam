## 1. Четыре правила (запомни)

1. **Перед работой** — `git pull`, затем открой **`00_СТАТУС.md`** (что уже сделано, какой этап сейчас).
2. **Делай только свой этап** по **`07_ПЛАН_РАБОТЫ.md`** — не перепрыгивай (например, BPMN до landscape).
3. **После работы** — обнови статус (или попроси агента), `git commit` + `git push` с понятным сообщением.
4. **Меняешь структуру папок** (создал, переименовал, удалил папку или ключевой файл) — **в той же задаче** обнови **`README.md`** в корне курсача и в затронутой подпапке; при необходимости — карту файлов в этой инструкции. Агент делает это автоматически по правилу в `.cursor/rules/kursovaya-modelirovanie-org.mdc`.

## 2. С чего начинать любую задачу

### Человек

1. `git pull`
2. `00_СТАТУС.md` → свой этап в `07_ПЛАН_РАБОТЫ.md`
3. Открыть **шаблон/файл из таблицы ниже**
4. Работать **внутри структуры папок**, не в личных Google Docs без синхронизации в репо

### Cursor / ИИ-агент

Скопируй в чат:

```
Работаем в курсовой «Моделирование организации» (Dodo Pizza).
Прочитай 00_СТАТУС.md и 07_ПЛАН_РАБОТЫ.md — раздел моего этапа [N].
Задача: [одна конкретная задача].
Не выдумывай факты о Dodo — только источники из 01_источники/ или [ТРЕБУЕТСЯ ИСТОЧНИК].
Не меняй рамку проекта (01_РАМКА_ПРОЕКТА.md) без согласования команды.
По окончании обнови 00_СТАТУС.md.
```

Агент в этом репо **уже знает** правила из `.cursor/rules/kursovaya-modelirovanie-org.mdc` — не отключай их.

## 4. Жёсткие запреты (иначе завалим защиту)

| Нельзя | Почему |
|--------|--------|
| Выдумывать факты, цифры, регламенты Dodo | Нет инсайдеров; только открытые источники + `assumptions_register.md` |
| Писать в гл. 1 «проблема в том, что…» | Гл. 1 — только справочные факты (методичка) |
| Два равных BPMN на несвязанные процессы | Один главный BPMN (время доставки), рекламации — упрощённая TO BE |
| TO BE на landscape как «уже внедрено» | Landscape нейтральный; улучшения — только в BPMN TO BE |
| Генерировать главу «под копипаст» без своих тезисов | Текст сдачи — синтез команды |
| Выдуманные DOI и книги в библиографии | Проверка по URL/издательству |
| Работать в параллельной копии без push | Команда потеряет контекст |

**Маркер в тексте:** нет источника → `[ТРЕБУЕТСЯ ИСТОЧНИК]` или строка в assumptions register.

## 5. Сюжет проекта (чтобы не расползлось)

**Один курсач, две связанные проблемы:**

1. **Доступность и срок доставки**: ближайшая точка может быть недоступна/перегружена, а срок доставки — неясен или срывается → главный BPMN, TO BE: альтернативная точка, пересчёт срока, уведомления.
2. **Рекламации** после плохой доставки → упрощённая TO BE: ИИ-помощник + человек.

Цепочка: *доступность/срок → срыв ожиданий → недовольство → жалоба*. Подробно: `03_РАНЖИРОВАНИЕ.md`, `01_РАМКА_ПРОЕКТА.md`.

## 6. Git: как не потерять наработки

```powershell
cd "d:\User\Desktop\работа\Cursor. Общий проект\02_Учёба\04_Курсач_по_моделькам"
git pull
# ... работа ...
git add .
git commit -m "Этап N: кратко что сделано"
git push
```

**Перед началом дня — pull. После задачи — push.** Конфликт → не затирать чужое; спросить в чате команды.
